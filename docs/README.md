the AI Development Constitution — approved and final. Implementation must
never contradict them; open each as a static HTML file in a browser to read.

|---|---|---|
| 1 | [phase-1-product-foundation.html](./phase-1-product-foundation.html) | Vision, personas, UX journey, design system, tech stack |
| 2 | [phase-2-customer-experience.html](./phase-2-customer-experience.html) | Every customer-facing screen, state, and interaction |
| 3 | [phase-3-owner-console.html](./phase-3-owner-console.html) | The owner's operating workflows — queue, payments, inventory |
| 4 | [phase-4-engineering-blueprint.html](./phase-4-engineering-blueprint.html) | Architecture, database schema, RLS, APIs, security, roadmap |

When code and docs disagree, the docs win — file a change against the
relevant phase document rather than silently drifting the implementation.
